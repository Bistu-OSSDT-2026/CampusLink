## 第3页：项目背景与需求

```
🎯 项目背景

  🏫 校园丢物痛点：
    • 物品丢失后难以找回
    • 信息发布渠道分散
    • 缺乏统一管理平台

  📋 功能需求：
    • 用户管理：注册、登录、个人中心、身份切换
    • 失物发布：信息填写、图片上传、违规校验
    • 信息检索：关键词搜索、分类筛选、时间排序
    • 交流互动：评论、私信、举报反馈
    • 管理员后台：审核、下架、处理举报
    • 亮点功能：智能匹配、数据可视化、消息提醒
```

## 第4页：技术选型

```
🛠️ 技术选型

  后端：
    • Python 3.8+
    • Flask 2.0+ 轻量级Web框架
    • Werkzeug 密码安全

  前端：
    • HTML5 + CSS3
    • Jinja2 模板引擎
    • Chart.js 数据可视化

  数据存储：
    • JSON 本地文件存储
    • 线程安全读写锁
    • 轻量化无数据库架构

  部署：
    • Flask 开发服务器
    • 浏览器访问 http://localhost:5000
```

## 第7页：数据存储设计

```
📊 数据模型

  User 用户模型：
    user_id, username, password_hash
    email, phone, role, nickname
    created_at, bio

  LostItem 物品模型：
    item_id, user_id, item_name, item_type
    description, location, contact_name
    contact_phone, report_time, status
    is_verified, is_deleted

  Comment 评论模型：
    comment_id, item_id, user_id
    content, create_time, is_deleted

  Message 消息模型：
    message_id, sender_id, receiver_id
    content, create_time, is_read, message_type

  Report 举报模型：
    report_id, reporter_id, item_id
    reason, detail, create_time
    status, handle_time, handle_result

  存储方式：JSON文件 + 线程锁
```
