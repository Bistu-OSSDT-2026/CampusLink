# 校园失物招领平台 - 需求规格说明书

## 1. 项目概述

### 1.1 项目背景

在校园生活中，学生和教职工经常会遗失物品。目前校园内缺乏一个统一的失物招领信息发布和查询平台，导致失物信息传播效率低下，很多遗失物品无法及时被找回。

### 1.2 项目目标

开发一款桌面端校园失物招领应用，提供失物信息的发布、查询、管理功能，帮助失主快速找回遗失物品，提高校园失物招领效率。

### 1.3 目标用户

- 在校学生
- 教职工
- 校园安保人员

## 4. 数据模型

### 4.1 失物信息（LostItem）

| 字段名 | 类型 | 含义 | 约束 |
|--------|------|------|------|
| item_id | String | 物品唯一标识 | 8位UUID |
| item_name | String | 物品名称 | 非空 |
| item_type | String | 物品类型 | 下拉选项 |
| description | String | 详细描述 | 可选 |
| location | String | 发现地点 | 非空 |
| contact_name | String | 联系人姓名 | 非空 |
| contact_phone | String | 联系电话 | 11位手机号 |
| report_time | String | 发布时间 | 自动生成 |
| status | String | 状态 | lost/found |
| image_path | String | 图片路径 | 可选 |

## 6. 技术架构

### 6.1 架构风格

采用分层架构（Layered Architecture）：

- **UI层**：负责用户界面展示和交互
- **Service层**：负责业务逻辑处理
- **Model层**：负责数据模型定义
- **Data层**：负责数据持久化（JSON文件）

### 6.2 模块划分

| 模块 | 职责 | 文件 |
|------|------|------|
| UI模块 | 主窗口、对话框 | `src/ui/` |
| Model模块 | 数据模型 | `src/models/` |
| Service模块 | 数据CRUD服务 | `src/services/` |
| Config模块 | 配置常量 | `src/utils/` |

### 6.3 数据流

```
用户操作 → UI层 → Service层 → JSON文件
JSON文件 → Service层 → UI层 → 用户展示
```
