## 1. 概述

校园失物招领平台是一款桌面端应用，旨在帮助校园内失物招领信息的发布与查询，方便失主找回遗失物品。

### 主要功能

- 发布失物信息
- 搜索失物信息
- 标记物品已找到
- 删除失物记录
- 查看物品详情

## 2. 安装与启动

### 2.1 环境要求

- 操作系统：Windows 10/11、macOS 10.15+、Linux Ubuntu 20.04+
- Python 版本：3.8 或更高版本

### 2.2 安装步骤

1. 下载项目文件到本地
2. 打开终端，进入项目目录

```bash
cd campus_lostandfound
```

3. 安装依赖（如果需要运行测试）

```bash
pip install pytest
```

### 2.3 启动应用

方式一：使用 Python 命令

```bash
python src/main.py
```

方式二：使用运行脚本（Linux/macOS）

```bash
chmod +x run.sh
./run.sh
```

## 3. 主界面说明

启动应用后，您将看到主界面，包含以下部分：

### 3.1 菜单栏

位于窗口顶部，包含三个菜单：

- **文件**：发布失物信息、退出应用
- **操作**：搜索、刷新列表
- **帮助**：使用帮助、关于应用

### 3.2 工具栏

位于菜单栏下方，包含常用操作按钮：

| 按钮 | 功能 |
|------|------|
| 发布失物 | 打开发布对话框，发布新的失物信息 |
| 搜索 | 打开搜索对话框，按条件搜索失物 |
| 刷新 | 重新加载失物列表 |
| 标记已找到 | 将选中的物品标记为已找到 |
| 删除 | 删除选中的失物记录 |

### 3.3 失物列表

位于界面中部，以表格形式展示所有失物信息：

| 列名 | 说明 |
|------|------|
| ID | 物品唯一标识 |
| 物品名称 | 遗失物品的名称 |
| 物品类型 | 物品所属类别 |
| 发现地点 | 物品发现的具体位置 |
| 联系人 | 发布信息的联系人 |
| 发布时间 | 信息发布的时间 |
| 状态 | 物品状态（丢失中/已找到） |

### 3.4 状态栏

位于窗口底部，显示当前记录数量和系统状态。

## 4. 功能操作指南

### 4.1 发布失物信息

1. 点击工具栏的"发布失物"按钮，或选择菜单"文件" → "发布失物信息"
2. 在弹出的对话框中填写以下信息：

| 字段 | 说明 | 必填 |
|------|------|------|
| 物品名称 | 遗失物品的名称 | 是 |
| 物品类型 | 从下拉列表中选择（电子产品、证件卡类等） | 是 |
| 详细描述 | 物品的特征、外观等描述 | 否 |
| 发现地点 | 物品发现的具体位置 | 是 |
| 联系人 | 您的姓名 | 是 |
| 联系电话 | 您的手机号码（11位） | 是 |

3. 点击"发布"按钮完成发布
4. 发布成功后会弹出提示，列表会自动刷新

### 4.2 搜索失物信息

1. 点击工具栏的"搜索"按钮，或选择菜单"操作" → "搜索"
2. 在弹出的对话框中设置搜索条件：

| 条件 | 说明 |
|------|------|
| 关键词 | 搜索物品名称或描述中包含的关键词 |
| 物品类型 | 按类型筛选 |
| 状态 | 按状态筛选（全部/丢失中/已找到） |
| 地点 | 搜索地点中包含的关键词 |

3. 点击"搜索"按钮执行搜索
4. 搜索结果会显示在主界面的列表中
5. 如需重新查看所有记录，点击"刷新"按钮

### 4.3 查看物品详情

1. 在失物列表中找到要查看的物品
2. 双击该物品所在的行
3. 系统会弹出详情窗口，显示物品的完整信息

### 4.4 标记物品已找到

1. 在失物列表中选中一条记录（点击行）
2. 点击工具栏的"标记已找到"按钮
3. 系统会将该物品的状态更新为"已找到"
4. 列表会自动刷新，状态列变为绿色的"已找到"

### 4.5 删除失物记录

1. 在失物列表中选中一条记录（点击行）
2. 点击工具栏的"删除"按钮
3. 系统会弹出确认对话框，请确认是否删除
4. 点击"是"确认删除，点击"否"取消操作
5. 删除成功后列表会自动刷新

### 4.6 刷新列表

点击工具栏的"刷新"按钮，或选择菜单"操作" → "刷新列表"，系统会重新加载所有失物记录。

## 5. 常见问题

### Q1: 应用启动后列表为空怎么办？

A: 这是正常现象。应用首次启动时没有任何数据，您需要点击"发布失物"按钮添加第一条记录。

### Q2: 搜索没有结果怎么办？

A: 请检查以下几点：
- 关键词是否正确
- 是否选择了正确的筛选条件
- 数据库中是否有符合条件的记录

点击"刷新"按钮可以查看所有记录，确认数据是否存在。

### Q3: 发布信息时提示电话格式错误？

A: 联系电话必须是11位数字的手机号码，请检查输入是否正确。

### Q4: 数据保存在哪里？

A: 数据保存在项目目录的 `data/lost_items.json` 文件中，所有信息均本地存储，不会上传到网络。

### Q5: 误删了记录能恢复吗？

A: 目前版本不支持记录恢复功能，请谨慎操作删除按钮。建议定期备份 `data/lost_items.json` 文件。

### Q6: 如何修改已发布的信息？

A: 当前版本暂不支持修改功能。如需修改，请先删除原记录，再重新发布一条新的信息。
