## 幻灯片3：项目背景

**问题描述**：
- 校园内失物招领信息传播效率低
- 缺乏统一的信息发布和查询平台
- 失主难以快速找回遗失物品

**解决方案**：
- 开发桌面端失物招领应用
- 提供信息发布、查询、管理功能
- 本地存储，无需联网即可使用

---

## 幻灯片4：项目目标

**核心目标**：
- 提高校园失物招领效率
- 帮助失主快速找回遗失物品
- 提供简洁易用的用户界面

**具体目标**：
- 支持失物信息发布
- 支持多条件搜索
- 支持状态管理（丢失中/已找到）
- 数据本地持久化存储

---

## 幻灯片7：技术架构

**架构风格**：分层架构

**模块划分**：
- UI层：负责界面展示和交互
- Service层：负责业务逻辑处理
- Model层：负责数据模型定义
- Data层：负责JSON文件存储

**技术栈**：
- Python 3.8+
- Tkinter（UI框架）
- dataclasses（数据模型）
- json（数据存储）
- pytest（测试框架）

## 幻灯片9：核心数据模型

**LostItem（失物信息）**：

| 字段 | 类型 | 说明 |
|------|------|------|
| item_id | String | 唯一标识（8位UUID） |
| item_name | String | 物品名称 |
| item_type | String | 物品类型 |
| description | String | 详细描述 |
| location | String | 发现地点 |
| contact_name | String | 联系人姓名 |
| contact_phone | String | 联系电话 |
| report_time | String | 发布时间 |
| status | String | 状态（lost/found） |

## PPT制作建议

1. **配色方案**：使用蓝色系为主色调，配合白色背景
2. **字体选择**：标题使用黑体，正文使用宋体或微软雅黑
3. **图片素材**：添加校园相关图片、应用截图
4. **动画效果**：适度添加过渡动画，增强演示效果
5. **时间控制**：建议总时长10-15分钟，每页讲解30-60秒
