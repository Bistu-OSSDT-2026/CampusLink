# 🏫 北京信息科技大学 - 校园失物招领平台 (CampusLink)

基于 **Flask** 框架开发的校园失物招领网页系统，采用暖橙活泼可爱风格设计。本项目为开源软件开发课程小组实践项目，适配北京信息科技大学校园场景。

## 📋 Bug 修复记录

### [已修复] 首页发布弹窗无响应问题

**问题类型**：Bug  
**出现位置**：首页失物信息发布弹窗  
**问题现象**：完整填写物品名称、地点、描述内容后点击提交按钮，按钮点击没有页面响应，帖子无法提交入库  
**原因分析**：

1. **缺少发布弹窗**：首页原本只提供跳转到 `/post` 独立页面的链接，没有内嵌发布弹窗，用户使用体验差
2. **缺少 AJAX 提交逻辑**：表单提交没有异步处理，页面刷新体验不佳
3. **数据路径依赖工作目录**：`JSONDataService` 使用相对路径 `'data'`，当运行目录不是项目根目录时会导致读写失败

**修复内容**：

| 文件 | 修改说明 |
|------|---------|
| `app/templates/items/index.html` | 添加发布失物弹窗（模态框）+ JS 异步提交逻辑，弹窗内显示错误/成功提示 |
| `app/blueprints/items_bp.py` | 添加 AJAX 请求检测（X-Requested-With 头），返回 JSON 响应，优化表单验证 |
| `app/services/data_service.py` | 数据路径改为基于项目根目录的绝对路径，修复运行时路径依赖问题 |

**验收结果**：

- ✅ 首页弹窗正常显示，包含完整的发布表单
- ✅ 点击提交按钮 AJAX 异步提交，弹窗不跳转页面
- ✅ 提交成功后弹窗显示"发布成功！"，自动关闭并刷新首页列表
- ✅ 验证失败时弹窗内显示红色文字提示（如"物品名称不能为空"、"手机号格式不正确"等）
- ✅ 未登录用户点击发布按钮自动跳转登录页
- ✅ 数据正确写入 `data/items.json`

## 🎯 功能模块

### 1. 用户管理模块
- 用户注册与登录
- 个人中心页面
- 修改个人信息
- 账号注销
- 身份切换（丢失者/捡拾者）

### 2. 失物发布模块
- 填写物品信息（名称、类型、描述、地点）
- 上传物品图片
- 违规信息校验拦截

### 3. 信息查询模块
- 关键词搜索
- 分类筛选（物品类型、状态）
- 发布时间排序

### 4. 交流互动模块
- 物品留言评论
- 用户私信功能
- 虚假信息举报反馈

### 5. 管理员后台模块
- 用户管理
- 信息审核
- 下架违规内容
- 处理举报

### 6. 亮点功能
- 智能匹配同类失物推荐
- 数据可视化统计图表（Chart.js）
- 新评论/私信/匹配消息提醒

## 🚀 快速开始

### 环境要求

- Python 3.8+
- pip 包管理工具

### 安装步骤

1. **克隆项目**
```bash
git clone <repository-url>
cd campus_lostandfound
```

2. **安装依赖**
```bash
pip install -r requirements.txt
```

3. **运行项目**
```bash
python run.py
```

4. **访问网站**

打开浏览器访问：http://localhost:8080

## 📖 使用说明

### 注册与登录
1. 点击右上角「注册」按钮
2. 填写用户名、密码、邮箱等信息
3. 选择身份（丢失者/捡拾者）
4. 注册成功后自动跳转登录页面

### 发布失物信息
1. 点击首页「发布失物」按钮，弹出发布弹窗
2. 填写物品名称、类型、描述等信息
3. 上传物品图片（可选）
4. 填写联系人信息
5. 点击「发布信息」完成提交
6. 提交成功后弹窗自动关闭，首页列表刷新

### 搜索失物
1. 在首页或搜索页面输入关键词
2. 选择物品类型和状态筛选
3. 选择排序方式（最新发布/名称排序）
4. 点击搜索查看结果

### 留言评论
1. 进入物品详情页面
2. 在评论框输入内容
3. 点击「发表评论」

### 私信联系
1. 在物品详情页面点击「私信联系」
2. 或在个人中心点击用户头像
3. 输入消息内容发送

### 举报功能
1. 在物品详情页面点击「举报」
2. 选择举报原因
3. 填写详细说明
4. 提交举报等待管理员处理
